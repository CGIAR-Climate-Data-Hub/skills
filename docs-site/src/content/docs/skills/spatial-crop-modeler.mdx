Orchestrates a full spatially-explicit crop yield simulation. You describe the area, crop,
and planting date. The skill checks your environment, generates a validated YAML config,
and runs DSSAT across every valid pixel in the datacube using `ag-cube-cm`.

## Two modes

| Mode | When to use | What happens |
|------|-------------|--------------|
| `with_cubes` | Weather and soil NetCDF files already on disk | Skip download — go straight to DSSAT |
| `full_pipeline` | Starting from scratch | Download CHIRPS + CHIRTS + AgERA5/NASA POWER + SoilGrids, build datacubes, then simulate |

## Climate sources

| Variable | Source | Notes |
|----------|--------|-------|
| Precipitation | CHIRPS | 0.05 deg, 1981–present, no API key |
| Max/Min temperature | CHIRTS | 0.05 deg, no API key |
| Solar radiation | AgERA5 | 0.1 deg — CDS key required at `~/.cdsapirc` |
| Solar radiation | NASA POWER | 0.5 deg — no API key, use when CDS unavailable |

## Supported crops and cultivars

| Cultivar | Crop | Best for |
|----------|------|----------|
| `IB1072` | Maize | Tropical / Central America / Sub-Saharan Africa |
| `PC0002` | Maize | Temperate regions |
| `IB1015` | Spring Wheat | Ethiopia, South Asia |
| `IB1487` | Winter Wheat | Europe, North Africa |
| `IB0001` | Bean / Soybean | General |

## Output

A NetCDF file with variable `HWAM` (harvest grain yield, kg/ha) and dimensions:

```
planting_window × year × y (latitude) × x (longitude)
```

After every run the skill reports:

```
pixels_ok      : 7   — DSSAT ran successfully
pixels_skipped : 18  — NaN weather or soil (expected at grid edges)
pixels_failed  : 0   — DSSAT runtime errors (investigate if > 5%)
mean HWAM      : 8782 kg/ha
```

Skipped pixels are normal — they occur where the coarser AgERA5 (0.1 deg) or SoilGrids
grids do not fully cover the finer CHIRPS (0.05 deg) grid edges.

## Install

```bash
pip install "ag-cube-cm[models] @ git+https://github.com/anaguilarar/ag-cube-cm.git"
pip install "aggeodata[download,mcp] @ git+https://github.com/anaguilarar/aggeodata.git"
```

:::caution[working_path must have no spaces]
DSSAT is a Fortran program that silently fails on paths with spaces.
Use `D:/dssat_work`, not `D:/OneDrive - CGIAR/...` for `working_path`.
Output NetCDF files can go anywhere.
:::

## AgERA5 setup (CDS key)

Required only when using AgERA5 for solar radiation. Register free at
[cds.climate.copernicus.eu](https://cds.climate.copernicus.eu/), then create `~/.cdsapirc`:

```
url: https://cds.climate.copernicus.eu/api/v2
key: <YOUR-UID>:<YOUR-API-KEY>
```

Use `rsds: nasa_power` in the config to skip this requirement entirely.

## Config schema reference

**with_cubes**

```yaml
mode: with_cubes

weather_path: D:/cubes/climate_hnd_2021_2021.nc
soil_path:    D:/cubes/soil_hnd.nc
output_path:  D:/results/yield_hnd.nc

simulation:
  country:      Honduras
  country_code: HND
  model:        dssat
  working_path: D:/dssat_work   # no spaces
  ncores:       2
  max_pixels:   null

crop:
  name:     Maize
  cultivar: IB1072

management:
  planting_date:        "2021-05-15"
  n_planting_windows:   1
  planting_window_days: 14
  fertilizer_schedule:  []
```

**full_pipeline**

```yaml
mode: full_pipeline

output_dir: D:/ag_cube_test/hnd_small
suffix:     hnd_small

spatial:
  bbox: [-87.5, 14.2, -87.2, 14.5]   # [xmin, ymin, xmax, ymax] WGS-84

dates:
  start: "2021-01-01"
  end:   "2021-12-31"

climate:
  sources:
    pr:     chirps
    tasmax: chirts
    tasmin: chirts
    rsds:   nasa_power   # or agera5 if CDS key available
  ncores:             2
  agera5_version:     "2_0"
  reference_variable: pr

soil:
  variables: [clay, sand, silt, bdod, cfvo, soc, phh2o, wv0010, wv0033, wv1500]
  depths:    ["0-5", "5-15", "15-30", "30-60", "60-100"]
  reference_variable: wv1500

simulation:
  country:      Honduras
  country_code: HND
  model:        dssat
  working_path: D:/dssat_work
  ncores:       2
  max_pixels:   null

crop:
  name:     Maize
  cultivar: IB1072

management:
  planting_date:        "2021-05-15"
  n_planting_windows:   1
  planting_window_days: 14
  fertilizer_schedule:  []
```
