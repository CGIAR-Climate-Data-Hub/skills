The soil counterpart to `climate-data-download`. You say which soil properties and
which area; the skill downloads SoilGrids rasters and optionally stacks them into a
single validated NetCDF cube ready for `spatial-crop-modeler` or
`geospatial-cube-processor`.

## What it covers

| Stage | Tool | Output |
|-------|------|--------|
| Download | `SoilGridsDownloader` (Python) or `aggeodata` soil-only YAML | One GeoTIFF per (variable, depth, statistic) under `soil_raw/` |
| Build cube | `SoilDataCubeBuilder` with explicit IGH source CRS → EPSG:4326 target | Single multi-variable NetCDF |
| Validate | Flat-format reshape · CRS bounds · variable completeness · climate-extent overlap | Cube ready for downstream tools |

## SoilGrids variables (default set)

| Variable | Description | Used for |
|----------|-------------|----------|
| `clay`, `sand`, `silt` | Texture (g/kg) | Soil hydrology, root growth |
| `bdod` | Bulk density | DSSAT, root penetration |
| `cfvo` | Coarse fragments | Effective soil volume |
| `soc` | Soil organic carbon (dg/kg) | Fertility |
| `phh2o` | pH (H₂O) | Crop suitability |
| `wv0010`, `wv0033`, `wv1500` | Volumetric water content at 10 / 33 / 1500 kPa | Field capacity, wilting point |

Depths (all in cm): `0-5`, `5-15`, `15-30`, `30-60`, `60-100`.

## The two non-obvious gotchas

1. **CRS** — SoilGrids serves data in Interrupted Goode Homolosine. `SoilDataCubeBuilder`'s
   default `crs="ESRI:54052"` produces broken extents. **Always pass
   `crs="+proj=igh +lat_0=0 +lon_0=0 +datum=WGS84 +units=m +no_defs"` and
   `target_crs="EPSG:4326"` explicitly.**
2. **`reference_variable`** — when using the YAML pipeline, `GENERAL.reference_variable`
   must be a non-null soil layer name. Default: **`sand`**. Setting it to `null` raises
   `reference_variable: non-null string required`.

## Output

A NetCDF cube (`soil_<country>.nc`) with dimensions `(y, x, depth)` per variable, in
EPSG:4326. The skill always validates it with three checks (CRS bounds, flat-format
reshape if needed, variable completeness) before returning.

**Troubleshooting**

| Symptom | Fix |
|---------|-----|
| `reference_variable: non-null string required` | Set `GENERAL.reference_variable: sand` in the YAML |
| Cube coordinates are huge numbers (e.g. `y = 1500000`) | Pass `crs=IGH` and `target_crs="EPSG:4326"` to `SoilDataCubeBuilder` |
| `no valid index for a 0-dimensional object` downstream | Cube is in flat format — run `reshape_flat_soil_cube` then re-save |
| Downloads time out / 403 | Wait and retry — `SoilGridsDownloader` resumes existing tiles |
